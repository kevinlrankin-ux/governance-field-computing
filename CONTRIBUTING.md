# Contributing

Governance Field Computing requires disciplined contribution.

Before proposing a change, identify whether the contribution is:

- definition
- schema
- test case
- evidence note
- architecture note
- methodology note
- example
- implementation code

Every contribution should include:

1. the component being changed,
2. the reason for the change,
3. evidence or hypothesis status,
4. ambiguity notes,
5. tests or examples if applicable.

Do not silently rewrite foundational artifacts. Create a new versioned file or changelog entry.
